This is the TLDR of the terms

  1. There are many input types
    • The 'type' parameter inside the input tag specifies what type of input you want
    • The most common ones are text,email, password, submit, radio
  2. The 'placeholder' parameter adds greyed out text inside the textbox
  3. The 'name' parameter helps the receiving webpage to identify each user input is being sent
  4. Label helps accessibility software to fill in the forms
  5. To link the label tag to the input tag:
    • Within the label tag, use the 'for' parameter
    • Within the input tag, use the 'id' parameter
  6. A form has 2 attributes
    1. Action: Where to send the data to (which website)
    2. Method: Which method to use? A HTTP GET or POST?
  7. It doesn't matter if you use input type submit or button element, it does the same job
    • However, if you use a button element, it has to be the last element in the form for it to work
  8. To use form validation, add the word required inside an input tag. See last input form to see how it works
  9. To set minimum length to passwords, use the pattern parameter inside the input element. pattern=".{8,12}"
  10. If you want to space out the form into different lines, you can envelop elements in div tags

To find what type of inputs you want to use, search for MDN inputs

Remember that within a form, there's two attributes

  1. Action: Where the form sends the data to
  2. Method: What HTTP method to use? GET or POST
If you create a form without an action attribute, it will send the data to the page where you are now If you create a form without a HTTP method, it will default to a GET request It doesn't matter if your form uses a button tag or an input button tag. both works
